# Howlingual UX/動作上の問題点 - 修正案

日付: 2026-02-04
範囲: 静的コードレビューのみ（実行検証なし）

1. `Esc` が常にウィンドウ全体を非表示にするため、設定/履歴/モーダルを閉じる意図の `Esc` が効かず突然隠れる。
修正案: グローバルの keydown を削除するか条件付きにする。`handleWindowKeydown` に一本化し、まずオーバーレイを閉じ、何も開いていない場合のみコンパクトモードでウィンドウ非表示にする。必要なら `Esc` の挙動設定を追加。
対象: `src/routes/+page.svelte:159`, `src/routes/+page.svelte:1115`。
テスト: 設定/履歴/モーダル表示中に `Esc` を押し、オーバーレイのみ閉じること。コンパクトモードではウィンドウが隠れること。

2. macOS 権限 UI が `check_permissions`/`request_permissions`/`open_screen_recording_settings`/`open_accessibility_settings` を呼ぶが、Tauri 側にコマンド登録がない。
修正案: 上記コマンドを Tauri で実装・登録するか、未対応ビルドでは UI を非表示にする。`invoke_handler` に登録を追加。
対象: `src/routes/+page.svelte:2041`, `src/routes/+page.svelte:5116`, `src-tauri/src/lib.rs:1500`。
テスト: 権限状態が表示され、ボタンでシステム設定が開くこと。画面収録が許可されるまで OCR をブロックすること。

3. クリップボードを一度空にして復元する実装が「テキストのみ」なので、画像/ファイルなどが失われる。
修正案: フル復元は実装コストが高いので、現実的には「操作前の警告表示」や「設定でON/OFF」を追加。将来的な理想案として UI Automation / Accessibility への移行を検討。
対象: `src-tauri/src/lib.rs:182`, `src-tauri/src/lib.rs:495`。
テスト: 警告を確認しない限り処理が進まず、設定OFF時はクリップボード改変が行われないこと。

4. ショートカット更新で旧ショートカットを先に `unregister` するため、新規登録が失敗すると旧ショートカットが消える。
修正案: 新規登録を先に行い、成功後に旧ショートカットを解除。API制約がある場合は失敗時に旧ショートカットを再登録。
対象: `src-tauri/src/lib.rs:558`。
テスト: 無効なショートカットを入力しても旧ショートカットが使えること。

5. 自動言語判定が簡易すぎて中国語が日本語判定になりやすく、ターゲット自動切替が誤作動。
修正案: 中国語/韓国語の簡易判定を追加、または十分な文字量がある場合のみバックエンドの検出結果を採用。自動切替は検出結果が安定してから実行。
対象: `src/routes/+page.svelte:174`, `src/routes/+page.svelte:1603`, `src/routes/+page.svelte:2507`。
テスト: 中国語入力時に検出言語と自動切替が正しいこと。

6. 言語選択リストと「自動検出」が日本語固定で、UI言語が変わってもローカライズされない。
修正案: 言語コードを保持して `getTargetLanguageName` で表示名を切り替える。`i18n` で「自動検出」もローカライズ。
対象: `src/routes/+page.svelte:1548`, `src/routes/+page.svelte:3748`。
テスト: UI言語を英/中/韓に切替え、言語リストがローカライズされること。

7. Linux ではコピー/貼り付けのシミュレーションが未実装なのに UI は成功したかのように見える。
修正案: Linuxは環境差が大きいので「未対応なら無効化」を基本にする。X11/Wayland判定で対応可なら実装を追加、不可なら説明を表示。
対象: `src-tauri/src/lib.rs:473`, `src-tauri/src/lib.rs:495`, `src/routes/+page.svelte:1675`。
テスト: Linux で「置換」操作が未対応なら明示され、成功表示が出ないこと。

8. 権限状態の初期値が「許可済み」になっており、実際の状態が出るまで誤表示。
修正案: 初期値を `false` か `null`（不明）にし、チェック中は「確認中…」を表示。
対象: `src/routes/+page.svelte:2041`。
テスト: macOS で設定を開いた直後に「確認中…」→正しい状態に切り替わること。

9. 権限 UI が使う `i18n` キーが未定義で、キー名がそのまま表示される。
修正案: `permissions`/`screenRecording`/`accessibility`/`granted`/`denied`/`grant` を `i18n.ts` に追加。
対象: `src/routes/+page.svelte:5120`, `src/lib/i18n.ts:5`。
テスト: 全言語で権限ラベルが正しく表示されること。

10. `startOCR` 失敗時に `isWaitingForOCR` が戻らず、以降の OCR イベント処理が誤作動する可能性。
修正案: `catch` で `isWaitingForOCR = false` を明示し、必要なら再試行 UI を追加。
対象: `src/routes/+page.svelte:1258`。
テスト: OCR 起動失敗時に状態がリセットされること。

11. OCR 開始で 300ms の固定待機に依存しており、環境次第でウィンドウが写り込む可能性。
修正案: ウィンドウ非表示の完了待ち（状態チェック）に切替え、プラットフォームごとに適応的に待機。
対象: `src-tauri/src/lib.rs:1040`。
テスト: OCR を繰り返してもキャプチャにウィンドウが写らないこと。

12. OCR クロップ画像を常にテンポラリ保存しており、プライバシー/ディスク面で問題。
修正案: デバッグ時のみ保存するフラグを設けるか、設定で任意化。リリースでは無効化。
対象: `src-tauri/src/lib.rs:1242`。
テスト: リリースビルドで一時ファイルが作成されないこと。

13. 入力テキストをクリアしても翻訳結果が残り、空状態が出ない。
修正案: クリア時に `translations`/`detailedExplanation`/`errorMessage` も初期化する。もしくは空状態判定を `inputQuery` ベースに変更。
対象: `src/routes/+page.svelte:3856`, `src/routes/+page.svelte:4224`。
テスト: クリアで空状態に戻ること。

14. 翻訳案数設定がクイック適用時のプレフィルとスケルトン件数に反映されない。
修正案: `translationCount` に合わせてスロット/スケルトンを生成。
対象: `src/routes/+page.svelte:1389`, `src/routes/+page.svelte:4337`。
テスト: 翻訳案数を1/2に設定した時、UIも同数になること。

15. コンパクトモードで「もっと見る」後に「折りたたむ」がない。
修正案: 「折りたたむ」操作を追加し `isStackExpanded` を戻せるようにする。
対象: `src/routes/+page.svelte:3418`。
テスト: 展開後に再度折りたためること。

16. お気に入りが `sourceText` で重複排除されるため、同一原文の別訳を複数保存できない。
修正案: 重複判定を `id` のみにするか、`translations` をソートしてからハッシュ化する。順序差分で別物扱いにならないようにする。
対象: `src/routes/+page.svelte:2126`。
テスト: 同一原文の異なる訳文を複数保存できること。

17. 履歴読み込み時に `isAutoDetect` や `detectedLang` が正しく復元されない。
修正案: 履歴に `isAutoDetect`/`detectedLang`/`sourceLang` を保存し、`loadHistory` で明示的に復元。既存履歴は未定義フィールドをデフォルト値で補完するマイグレーションを用意。
対象: `src/routes/+page.svelte:2200`。
テスト: 自動検出の履歴を読み込んだ際に同状態が再現されること。

18. クリップボードコピーは `await` せず、失敗しても成功表示が出る。
修正案: `await navigator.clipboard.writeText` して失敗時はエラー表示/成功演出をしない。
対象: `src/routes/+page.svelte:1666`。
テスト: クリップボード拒否時に成功表示が出ないこと。

19. 「起動時はメイン画面を最小化」は実際には「非表示」で、文言と挙動がズレる。
修正案: 文言を「起動時は非表示」に変更するか、実際に最小化処理を実装。必要ならトレイ通知を追加。
対象: `src/routes/+page.svelte:396`。
テスト: 設定の文言と動作が一致すること。

20. 閉じる操作が常に「隠す」挙動で、UI上の終了導線が見えない。
修正案: 設定/情報タブに「終了」ボタンを追加、またはトレイメニューで明示。初回のみ説明ダイアログも検討。
対象: `src-tauri/src/lib.rs:1543`。
テスト: OSの強制終了なしで終了できること。

21. アプリバージョンがハードコードされている。
修正案: Tauri から `app.get_version()` を取得するコマンドを用意しUIに反映。もしくは build 時に `package.json` から注入。
対象: `src/routes/+page.svelte:45`。
テスト: バージョン更新時にUI表示も更新されること。

22. APIキーが `localStorage` に平文保存される。
修正案: 現実的には「保存時の警告表示」や「記憶する」トグルを追加する。将来的に Tauri の安全ストレージプラグイン検証の上、OSキーチェーン対応を検討。
対象: `src/routes/+page.svelte:780`。
テスト: 初回保存時に警告が表示され、保存OFFならキーが残らないこと。

23. TTS が未対応の環境で無反応になり、失敗理由が伝わらない。
修正案: `speechSynthesis` 非対応時はボタンを無効化し、ツールチップ等で理由を提示。
対象: `src/routes/+page.svelte:1720`。
テスト: 未対応環境でUIが明確に無効状態になること。
