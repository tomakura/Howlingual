Tauri + Rust を用いて、Windows/macOS対応の常駐翻訳/解説アプリをゼロから実装してください。UIはモダンで綺麗かつ軽量、操作がリッチ（アニメーション多め）なアプリを作成。

https://codewiki.google にアクセスして、このプロジェクトのドキュメントを参照しながらコーディングを行ってください。

# 目的（ユーザー体験）
- 任意アプリでテキストを選択
- グローバルショートカットで本アプリが即表示（ポップアップ/オーバーレイ）
- 選択テキストを取得してAI APIで:
  1) 翻訳（候補を複数）
  2) 解説（なぜその訳にしたか：語法/ニュアンス/文脈・代替案の理由）
  を表示
- さらに「範囲選択 → OCR」で画像内文字も取り込み可能
- 通常起動して手入力でも同機能を使える（メイン画面に入力欄）
- 翻訳候補・原文をワンクリックでOS標準TTSで読み上げできる

# 技術スタック（指定）
- Tauri（Rustバックエンド）
- フロントは Svelte + TypeScript + Vite
- 軽量で高速。
- UIは i18n 対応（JP/EN切替）
- 初期言語はシステム言語を参照して設定
- 設定保存はローカル（JSON/SQLiteいずれか）
- APIキー等の秘匿情報はOS資格情報ストア（macOS Keychain / Windows Credential Manager）に保存する。平文でディスク保存しない。

# 重要なデザイン制約（必須）
- グラデーションは最小限
- ぼかし（blur / backdrop-filter）は許容
- アニメーションは「全体的に」付けてリッチにする（遷移・表示・ホバー・押下・リスト更新など）
  - ただし60fpsを意識し、重いアニメは控える
  - OS/ユーザー設定の「prefers-reduced-motion」およびアプリ設定でアニメ抑制も可能（アクセシビリティ）

# コア機能要件（必須）
## A. 常駐 + グローバルショートカット
- 常駐（トレイ/メニューバー）を実装
- デフォルトショートカット例:
  - 翻訳起動: Win=Ctrl+Shift+T / Mac=Cmd+Shift+T
  - OCRモード: Win=Ctrl+Shift+O / Mac=Cmd+Shift+O
- 設定画面でショートカット変更可能（競合時は警告）
- ショートカット押下で:
  - まずウィンドウを即表示（体感軽く）
  - その後に取得/推論を開始し、結果が来たら更新

## B. 選択テキスト取得（テキスト選択 → 起動）
- 推奨方式: クリップボード方式
  1) 現在のクリップボードを退避
  2) 前面アプリへ Cmd/Ctrl+C を擬似送信
  3) クリップボードから文字列取得
  4) 退避分を復元
- 例外・失敗時:
  - 取得不可の場合は「取得できませんでした」を表示し、手入力欄へフォーカス
  - クリップボード復元は必ず保証（panic/例外でも復元するよう構造化）
- macOSでは必要に応じて権限（アクセシビリティ等）の案内を初回導線で表示

## C. 範囲選択 + OCR（必須）
- OCRモードのショートカットで、画面全体の「範囲選択オーバーレイ」を表示
  - 半透明の全面ウィンドウ
  - ドラッグで矩形選択
  - ESCでキャンセル
- 選択範囲をスクリーンショットしてOCRを実行し、得られたテキストを入力として翻訳/解説へ送る
- OCR言語はJP/EN（自動判定または選択式。設定でデフォルト言語を指定可能）
- macOSはScreen Recording等の権限が必要な場合、分かりやすい導線とチェックUIを実装
- OCR結果の後処理:
  - 改行/ハイフネーション/余計な空白を整形する
  - 失敗時は画像プレビューと再OCRボタンを提供
- プライバシー: スクショ画像は原則メモリ上で扱い、ディスクへ永続保存しない（必要なら一時ファイルは即削除）

## D. AIプロバイダ切替（OpenAI / Google / Claude）+ モデル選択（必須）
- 設定画面でプロバイダを切替可能
- 各プロバイダごとに:
  - APIキー入力（資格情報ストアに保存）
  - モデル選択（ドロップダウン + 手入力も許可）
  - 温度 / 最大トークン / タイムアウトなど基本パラメータ
  - 疎通テストボタン（成功/失敗理由を表示）
- プロバイダ抽象化を行い、UIは統一インターフェースで呼べること

## E. 出力仕様（翻訳候補 + 解説）
- 入力: 選択テキスト / OCRテキスト / 手入力テキスト
- 出力:
  - 翻訳候補を複数（例: 3〜5）
  - 各候補に「短い理由（1〜2行）」を付ける
  - 別セクションに「詳しい解説（なぜそう訳したか、他候補を採らなかった理由、語感/敬意/硬さ等）」を表示
- 操作:
  - 候補ごとにコピー / お気に入り / ピン留め / 再生成
  - 入力と出力の履歴（直近N件、検索、個別削除、全削除）

## F. トーン（スタイル）選択：3個以上組み合わせ可（必須）
- 「文体」ではなく「トーン/スタイル」として設計し、UIもその用語に統一する
- 初期プリセット（無難で一般的な名称）:
  - フォーマル
  - カジュアル
  - 簡潔（短く）
  - 丁寧（やわらかい）
  - 技術（正確・用語重視）
  - 学習（初学者向けに噛み砕く）
  - クリエイティブ（表現豊か）
  - カスタム（ユーザーが指示文を自由記入）
- 各スタイルは強度3段階（1=弱, 2=中, 3=強）
- スタイルは「3個以上」組み合わせ可能
  - UIはチップ/リストで複数追加、並び替え、削除可能
  - 例: フォーマル(2) + 簡潔(1) + 技術(2)
- スタイルからのプロンプト生成はテンプレート化し、保守性を確保する
- 攻撃的/差別的になりやすい口調にならないガード（安全フィルタ）を入れる

## G. 画面構成（必須）
### メイン画面
- 入力欄（手入力）
- スタイル（複数 + 強度）セレクタ
- 翻訳候補リスト
- 詳細解説
- 履歴（検索、削除）
- JP/EN切替トグル（または設定から）
- 音声読み上げ（TTS）操作（候補/原文）

### 設定画面（タブ）
1) Provider
   - OpenAI/Google/Claude切替
   - APIキー入力（安全保存）
   - モデル選択（ドロップダウン + 手入力）
   - パラメータ
   - 接続テスト
2) Appearance
   - Light/Dark/System
   - フォントサイズ（小/中/大）
   - アニメーション（ON/控えめ/OSに従う）※prefers-reduced-motionも尊重
   - グラデーション禁止が守られていること（開発時チェック）
3) Shortcuts
   - 翻訳起動ショートカット
   - OCRショートカット
   - 常駐有効/無効
4) OCR
   - 言語（JP/EN/Auto）
   - 前処理（しきい値/コントラスト等：簡易でOK）
5) Speech / Audio（TTS）
   - TTS方式: OS標準（既定）/ API（実装のみ、既定では無効）
   - 音声（Voice）選択（JP用/EN用で別々に持てる）
   - Rate（速度）/ Pitch（可能なら）/ Volume
   - 自動再生（結果表示後に読む）ON/OFF
   - 連続再生（候補1→2→3…）ON/OFF
   - テスト再生ボタン
6) Info
   - バージョン
   - 使い方（初回導線）
   - プライバシー（送信データ、ログ方針、スクショ扱い）
   - ライセンス表記

## H. 音声読み上げ（TTS）機能（必須）
- 翻訳候補および入力テキストに対して、ワンクリックで音声読み上げできる
  - 候補行ごとに「再生/一時停止/停止」を提供
  - 入力文（原文）にも読み上げボタン
- 言語（JP/EN）に応じて適切な音声を自動選択（設定で上書き可能）
- 再生中は該当候補をハイライトし、別候補を押したら前の再生を停止して切り替える
- エラー処理: 音声が見つからない/利用不可の場合は理由と対処をUIに表示

### H-1. OS標準TTS（本番必須）
- 追加APIキー不要で動作するOS標準TTSを実装する（Win/mac）
- 可能ならオフラインでも動く構成を優先する

### H-2. API経由TTS（“実装だけ”して封印：必須）
- API経由TTSは、将来解放されたときにONにできるよう「コードとUI枠は実装」する
- ただし既定では無効（feature flag / 実験機能）とし、一般ユーザーのUIからは隠すか無効化する
- エンドポイント/パラメータ/制約は codewiki.google の該当ドキュメントを参照して実装する
- APIキー管理は翻訳プロバイダとは独立して設計（将来別キーになる可能性を考慮）
- 未解放状態でもビルドが通り、アプリが正常動作すること（OS TTSは必ず動く）

# 非機能要件（必須）
- 速度: ショートカット押下→UI表示は即時。推論は非同期で更新
- エラー処理: ネットワーク失敗/レート制限/不正キー/タイムアウトを分かりやすく表示
- ログ: 入力文やOCR結果をログに残さないモード（デフォルトON推奨）
- セキュリティ: APIキーは資格情報ストアへ。設定ファイルにキーは出さない
- 国際化: UIはJP/EN切替。文言は一箇所で管理し、追加が容易な構造にする

# 実装の進め方（指示）
1) アーキテクチャとディレクトリ構成を提示
2) Tauriプロジェクト生成、最小UI + 常駐 + 2種類ホットキー（翻訳/OCR）を動かす
3) クリップボード方式の選択テキスト取得（復元保証）を実装
4) 範囲選択オーバーレイ + スクショ + OCR を実装
5) プロバイダ抽象化を実装し、OpenAIを先に通してからGoogle/Claudeを追加
6) スタイル（複数 + 強度）テンプレを実装し、翻訳候補+解説をJSONスキーマで返す
7) OS標準TTSを実装し、API経由TTSはfeature flagで封印した上で“実装だけ”入れる
8) 設定画面（タブ）、テーマ、i18n、ショートカット編集、資格情報保存を実装
9) 受け入れ基準チェック、手動テスト手順、README整備

# 出力形式（必須）
- リポジトリに含める主要ファイル一覧
- ビルド/実行手順（Win/mac）
- 手動テスト手順（選択→翻訳、範囲選択→OCR→翻訳、手入力→翻訳、プロバイダ切替、モデル切替、言語切替、TTS）
- 重要設計（プロバイダIF、スタイルテンプレ、セキュア保存、権限導線、TTS設計）
- 重要: UIにグラデーションが一切含まれないことを検査する方針（lint/検索/テスト等）も提示

# 受け入れ基準（必須）
- Win/macで起動し、常駐・ホットキー（翻訳/OCR）が動く
- 選択テキスト→候補+解説が表示される
- 範囲選択OCR→候補+解説が表示される
- 手入力でも同様に動く
- Provider切替とAPIキー安全保存が機能する（平文保存なし）
- Light/Dark/SystemとJP/EN切替が機能する
- UIにグラデーションが一切ない
- アニメーションが全体に入っていてリッチに感じる
- OS標準TTSで候補/原文を読み上げできる
- API経由TTSはコードとして存在するが既定では無効で、未解放でもアプリが壊れない
