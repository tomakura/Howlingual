## 作業内容（レビュー依頼）

以下の内容で、専用UIのモックアップ作成に着手します。ご確認をお願いします。

### 目的
- テキスト選択 + ショートカットで開く「専用画面」のUIモックアップ作成

### 反映方針
- メイン画面から不要要素を削除し、ウィンドウサイズを小さくする
- メイン画面を開くボタンを用意する
- 言語選択UIは現状維持
- 選択文章の置き換え機能を用意する
- 文体選択はミニマルにし、リスト表示形式へ変更する

### UI/UXで特に考慮する変更点
- 固定サイズの小型ウィンドウ（モックアップでは 420x520 を想定）
- ヘッダーは最小限（言語選択 + メイン画面を開くボタン + 閉じる）
- 選択テキストのプレビュー領域を上部に配置し、置き換えアクションへ直結
- 翻訳候補を最優先で表示し、解説は折りたたみや軽い区切りで負担を減らす
- 文体（トーン/スタイル）は縦リストで簡潔に表示、強度は行内で最小表示
- 操作はキーボード優先で使える導線（Enter/Copy/Replace等）を想定
- 状態（取得中/翻訳中/失敗）は軽いアニメーションで明示

全体的なUI、ボタン、アニメーションなどのデザインはメイン画面を参照し、できるだけ合わせて。

### 作業予定
- 画面レイアウトの整理（専用画面のモックアップ）
- UIコンポーネントの再構成（最小限の要素配置）
- 必要に応じたスタイル調整（サイズ、余白、視線誘導）

### 決定事項
- 専用画面のサイズ: 固定
- 「メイン画面を開くボタン」の配置: 任せる

レビュー後に着手します。
